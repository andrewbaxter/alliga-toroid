Merman Webview/Configurable Syntax

This is an example of the Block Webview widget for using Merman to embed snippets of code in a website.

Merman is an AST viewing/editing solution. It uses a syntax definition to provide a traditional source-like representation of an AST, but maintaining all of the structure under-the-hood. This is a shift from most editors which operate on linear text, and where the syntax is built into the compiler which uses it to translate the text into an AST internally.

One of the benefits of shifting the responsibility of syntax definition to the client is that users can modify the syntax to their liking and stay source-compatible with all other users. This example shows this feature off by casting a single ESTree (Javascript) snippet in Javascript clothes (looks nearly the same), Python clothes, and Lisp clothes. As a note, syntax customization would probably be mostly done in IDEs - for maximum readability people embedding code in websites with the webview should stick to official syntaxes.

  • I only implemented the portion of ESTree that was necessary for this example
  • Some things, like parentheses around broken expressions in Python, are there to make the parser easier to write/support the repl. This isn't necessary when working with an AST, so (again) there are some details where the syntax flavor diverges from the inspiration.

Original Source

The way I did this was I used Acorn to translate a small Javascript program into a JSON ESTree AST. The program I used is:

for (let i = 1; i <= 100; ++i) { let out = "" if (i % 3 === 0) { out += "Fizz" } if (i % 5 === 0) { out += "Buzz" } if (out.length === 0) { out += i } console.log(out + " ") }

(this is plain text :P)

The ESTree AST output is included at the end because it's on the huge side of things.

Javascript-like

I fed the AST directly into the Webview widget along with the Javascript-like syntax - Merman operates on ASTs directly. As you can see it produced a representation similar to the input source above. Highlighting defined in the syntax and are based on the structure (no regexes).

Python-like

This is mostly a demonstration of bracket-less syntax, rather than trying to make it look exactly like Python.

Python doesn't have any c-style for loop. I could have introduced a sugar to make "c-style for loops with integer bounds, 1-unit increment, less than condition" look like "enumerate(x)" but I thought that would be a bit awkward... it's not Python after all.

Prodel-like

Prodel is a Japanese literate programming language. This is unfortunately a poor immitation of the syntax described there. The for loop keywords don't make a ton of sense, also the mixture of full-width characters and space-less ascii is pretty ugly. I wanted to try vertical text but hit a number of browser/w3c spec issues - no way to get vertical text metrics, 「」 and ー don't get rotated 90 degrees.
